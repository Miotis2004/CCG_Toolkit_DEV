# Minimum Viable Vertical Slice

The first playable vertical slice should prove the toolkit can author cards, build a legal deck, start a deterministic match, play basic cards, resolve combat, and display the result in Unity UI.

## Acceptance Criteria by Future Phase

- **Phase 1:** Designers can create validated sample card definitions.
- **Phase 2:** Designers can browse, search, validate, and export card data.
- **Phase 3:** Edit-mode tests can run a deterministic match core without scene dependencies.
- **Phase 4:** Deck and collection data can be validated and serialized.
- **Phase 5:** At least one damage, draw, summon, and buff effect resolves through the event pipeline.
- **Phase 6:** Basic minion combat validates attacks, damage, and death processing.
- **Phase 7:** A local match scene reflects deterministic match events.
- **Phase 8:** A player can browse cards and save a sample deck.
